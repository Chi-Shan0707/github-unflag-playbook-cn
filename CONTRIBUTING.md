# 参与贡献

[English](CONTRIBUTING.en.md) / 中文

---

感谢你愿意为本仓库做出贡献。无论你是被 flag 的当事人、有帮助经验的人、还是只想帮忙整理信息的人，都欢迎。

---

## 如何贡献

### 1. 提交你的案例

如果你曾被 flag 并愿意分享经历：

1. 复制 [cases/template.md](cases/template.md)
2. 填写你的信息
3. 将文件重命名为 `<你的GitHub用户名>.md`
4. 提交 PR
5. 在 PR 中说明你同意公开这些信息

你也可以：
- 在 Issue 中填写信息（我们会帮你整理）

### 2. 加入存在档案

如果你只想被记录存在，不需要写完整案例：

1. 在 [Issue](../../issues) 中提交，标题加 `[存在档案]` 前缀
2. 只需提供：用户名 + 大概时间 + 一句话

### 3. 改进文档

如果你发现文档中有错误、过时信息、或可以改进的地方：

1. Fork 本仓库
2. 创建分支
3. 修改内容
4. 提交 PR

### 4. 参与讨论

在 [Discussion](../../discussions) 中分享你的经验、建议、或问题。

---

## 贡献准则

### 内容要求

- **基于事实**：所有信息应基于官方文档、社区案例、或个人真实经历
- **标明来源**：引用信息请注明出处
- **区分层级**：明确区分"官方确认"和"社区经验"
- **不提供违规方法**：不教人绕过规则或使用黑灰产手段

### 态度要求

- **尊重他人**：被 flag 的人可能正在经历焦虑和挫败，请保持善意
- **不指责受害者**：不要暗示"被 flag 就是你做错了"
- **保持客观**：即使是个人经历，也尽量陈述事实

### 隐私要求

- **不公开他人信息**：除非获得明确同意
- **脱敏处理**：工单编号、邮箱等敏感信息需要脱敏
- **可随时撤回**：任何人都可以随时要求删除自己的信息

## 双通道联系

考虑到被 flag 的人可能无法正常使用 GitHub 功能，我们提供两个联系渠道：

| 渠道 | 说明 |
|------|------|
| **GitHub 内部** | Issue / Discussion / PR |

## Issue 模板

### 提交案例

**标题**：`[案例] @你的用户名`

**内容**：复制 [cases/template.md](cases/template.md) 中的内容填写。

### 加入存在档案

**标题**：`[存在档案] @你的用户名`

**内容**：
```
- 用户名：
- 大概时间：
- 一句话：
```

### 内容修正 / 建议

**标题**：简洁描述你的修正或建议

**内容**：
- 涉及哪个文档
- 当前内容是什么
- 建议改为
- 理由或来源
